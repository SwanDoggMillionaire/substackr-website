## What Substackr Is

**Two products for Substack writers:**

1. **Essay Idea Engine (Free)** ‚Äî Enter your niche/topic (required) + optionally a writer you admire + your own handle. Get 5 essay ideas. If a writer is provided, ideas are inspired by their approach; if no writer, ideas are generated from niche alone. A topline writer overview is shown when a writer was researched; the full 7-section profile is locked behind Pro.

2. **Substack Audit (Pro)** ‚Äî Enter your own Substack handle. Get a full strategic analysis: all 7 profile sections, About page improvement tips, and a clickable list of your recent posts. Free during beta. Auth-gated in v2.

**Freemium gate pattern:** The full writer profile (7 sections) is always generated by Claude but shown as locked cards (padlock icon + teaser copy + "Unlock Pro ‚Üí") in the free tier. The full analysis is only visible at `/audit`.

**Promotion strategy:** Swanny promotes via his own Substack newsletter. His readers are mostly not existing Substackers ‚Äî they're AI/learning curious people considering starting a newsletter. The free tier works for anyone (no Substack required). The Pro tier is the upsell once they're writing.

**Writer mode only** ‚Äî no tab toggle, no self-analysis here.

**URL parameters:**
- `?niche=X` ‚Äî auto-starts with niche only (no writer research phase)
- `?niche=X&writer=Y` ‚Äî auto-starts with both
- `?niche=X&writer=Y&userHandle=Z` ‚Äî auto-starts with all fields
- Auto-start triggers on `initialNiche || initialWriter` (either is sufficient)

**State machine:** `'idle' | 'researching' | 'generating-ideas' | 'success' | 'error'`

**Auto-chaining flow:**
1. `window.scrollTo({ top: 0 })` immediately on search start
2. If writer provided: `setState('researching')` ‚Üí POST `/api/research` with `isSelf: false`

3. `setState('generating-ideas')` ‚Üí POST `/api/essay-ideas`
4. `setState('success')` ‚Äî shows essay ideas, then ProfileCard only if `profile.writerName` is set

**Mobile layout while loading:** form column hidden (`hidden lg:block`), loading state fills full width above fold.

**Results layout:**
- `EssayIdeas` component (ideas first)
- `ProfileCard` with `toplineOnly={true}` ‚Äî only rendered when a real writer was researched

**SearchForm:** Niche required (min 2 chars). Writer + handle optional. Suggested writers randomised from pool of 30 on each render ‚Äî clicking a suggestion auto-submits with the writer name (bypasses niche requirement).

## AI / Claude Architecture

### Research call (`researchWriter`)
- Model: `claude-sonnet-4-6`, `max_tokens: 2800`, `temperature: 0.3`
- When `substackData` provided: injected as "VERIFIED SUBSTACK DATA" block ‚Äî Claude instructed to treat as authoritative ground truth
- `SELF_ANALYSIS_ADDITION` appended when `isSelf === true` ‚Äî second-person tone, aboutMeTips field, about-page-focused gap analysis
- `extractJSON()` strips accidental code fences before `JSON.parse()`

### Essay ideas call (`generateEssayIdeas`)
- Model: `claude-sonnet-4-6`, `max_tokens: 2000`, `temperature: 0.7`
- Two clearly labelled blocks in the prompt:
  - `=== INSPIRATION WRITER: [name] ===` ‚Äî condensed profile summary (niche, positioning, content patterns, theGap)
  - `=== YOUR NEWSLETTER ===` ‚Äî either live Substack data (when userHandle provided) or typed niche
- `userDataSource` set on result: `'live' | 'niche' | 'none'`

### WriterProfile JSON structure
```
writerName, confidenceLevel (HIGH/MEDIUM/LOW), oneLiner
dataSource: 'live' | 'training'
researchProfile, nicheAndTopicFocus (+ bullets), audienceResonance
positioning, monetisationStrategy, contentPatterns
strategicSummary: { title, insights[3], theGap }
similarWriters[2-3]
isAnalysingSelf?: boolean
aboutMeTips?: string[]    ‚Äî only when isSelf=true
recentPosts?: RecentPost[] ‚Äî only when live data fetched
```

### Live Substack API
- About: `https://[handle].substack.com/api/v1/about` ‚Äî publication info, description, subscriber count
- Posts: `https://[handle].substack.com/api/v1/posts?limit=12` ‚Äî recent posts
- Both undocumented but stable public endpoints, no auth required
- `fetchSubstackData()` builds a `contextString` for Claude (injected as verified data) and a `posts` array for UI display
- Post URL pattern: `https://[handle].substack.com/p/[slug]`

## API Routes

### POST `/api/research`
- Input: `{ writerName: string, isSelf?: boolean }`
- Output: `WriterProfile` JSON (with `recentPosts` attached if live data fetched)
- Extracts handle from writerName ‚Üí fetches live Substack data if valid handle found
- Error codes: `INVALID_INPUT` (400), `PARSE_ERROR` (500), `API_ERROR` (500)

### POST `/api/essay-ideas`
- Input: `{ profile: WriterProfile, userNiche?: string, userHandle?: string }`
- Output: `EssayIdeasResult` JSON
- If `userHandle` provided: fetches user's own Substack data as `userData` ‚Äî separate from inspiration writer's profile
- `effectiveNiche` fallback: `userNiche || profile.writerName`

## Known Limitations / Open Issues

| Issue | Severity | Notes |
|-------|----------|-------|
| Freemium gate is client-side only | Medium | Full WriterProfile JSON is returned to the browser ‚Äî locked sections are presentational. Fine for beta; v2 gate is route-level (Clerk on `/audit`) |
| Full profile sent client‚Üíserver for essay ideas | Low | Client POSTs the full WriterProfile to `/api/essay-ideas`. Fine for now; in v2 store in Vercel KV by session |
| Small-writer profile quality | Low | Claude has limited training data on smaller writers. Live fetch helps significantly when handle is provided |
| Niche-only idea quality | Low | When no writer is provided, research phase is skipped and a minimal profile is built client-side. Claude has less to work with ‚Äî ideas are based solely on niche string. Acceptable for v1. |
| Anthropic quota shared with OpenClaw | Medium | Monthly spend cap hit once in March 2026. Monitor at console.anthropic.com |
| Analytics metadata thin | Low | Events fire but no latency tracking or error-type classification. Sufficient for early beta; extend before v2 |
| LoadingState message replacement fragile | Low | Uses `.replace('the writer', writerName)` ‚Äî breaks silently if a message is edited without the literal phrase. Low risk until message pool expands |
| Substack API partial failures silent | Low | If about fetch succeeds but posts fail, Claude gets partial data still marked as VERIFIED. No user-facing indication |
