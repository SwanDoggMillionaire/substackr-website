## What Substackr Is

**Two products for Substack writers:**

1. **Essay Idea Engine (Free)** — Enter a writer you admire + optional niche/handle. Get 5 essay ideas inspired by their approach, adapted for your newsletter. A topline writer overview is shown; the full 7-section research profile is locked behind Pro.

2. **Substack Audit (Pro)** — Enter your own Substack handle. Get a full strategic analysis: all 7 profile sections, About page improvement tips, and a clickable list of your recent posts. Free during beta. Auth-gated in v2.

**Freemium gate pattern:** The full writer profile (7 sections) is always generated by Claude but shown as locked cards (padlock icon + teaser copy + "Unlock Pro →") in the free tier. The full analysis is only visible at `/analyse`.

**Promotion strategy:** Swanny promotes via his own Substack newsletter. His readers are mostly not existing Substackers — they're AI/learning curious people considering starting a newsletter. The free tier works for anyone (no Substack required). The Pro tier is the upsell once they're writing.

**URL parameters:**
- `?writer=X` — auto-starts search
- `?writer=X&niche=Y&userHandle=Z` — auto-starts with all fields

**State machine:** `'idle' | 'researching' | 'generating-ideas' | 'success' | 'error'`

**Auto-chaining flow:**
1. `setState('researching')` → POST `/api/research` with `isSelf: false`
2. On success: `setState('generating-ideas')` → POST `/api/essay-ideas`
3. On both complete: `setState('success')` — shows essay ideas first, then topline ProfileCard

**Results layout:**
- `EssayIdeas` component (ideas first)
- `ProfileCard` with `toplineOnly={true}` — shows header + niche bullets + 7 locked sections with padlock + teaser

**SearchForm:** Writer name required (min 2 chars). Niche + handle optional. Suggested writers randomised from pool of 30 on each render.

### `/analyse` — Substack Audit (Pro tier, free in beta)
`app/analyse/page.tsx` — `'use client'`, wrapped in `<Suspense>`.

**URL parameters:**
- `?handle=X` — auto-starts analysis

**State:** `'idle' | 'loading' | 'success' | 'error'`

**Flow:**
1. POST `/api/research` with `isSelf: true`
2. On success: shows full `ProfileCard` (all 7 sections, aboutMeTips, recentPosts)

## AI / Claude Architecture

### Research call (`researchWriter`)
- Model: `claude-sonnet-4-6`, `max_tokens: 2800`, `temperature: 0.3`
- When `substackData` provided: injected as "VERIFIED SUBSTACK DATA" block — Claude instructed to treat as authoritative ground truth
- `SELF_ANALYSIS_ADDITION` appended when `isSelf === true` — second-person tone, aboutMeTips field, about-page-focused gap analysis
- `extractJSON()` strips accidental code fences before `JSON.parse()`

### Essay ideas call (`generateEssayIdeas`)
- Model: `claude-sonnet-4-6`, `max_tokens: 2000`, `temperature: 0.7`
- Two clearly labelled blocks in the prompt:
  - `=== INSPIRATION WRITER: [name] ===` — condensed profile summary (niche, positioning, content patterns, theGap)
  - `=== YOUR NEWSLETTER ===` — either live Substack data (when userHandle provided) or typed niche
- `userDataSource` set on result: `'live' | 'niche' | 'none'`

### WriterProfile JSON structure
```
writerName, confidenceLevel (HIGH/MEDIUM/LOW), oneLiner
dataSource: 'live' | 'training'
researchProfile, nicheAndTopicFocus (+ bullets), audienceResonance
positioning, monetisationStrategy, contentPatterns
strategicSummary: { title, insights[3], theGap }
similarWriters[2-3]
isAnalysingSelf?: boolean
aboutMeTips?: string[]    — only when isSelf=true
recentPosts?: RecentPost[] — only when live data fetched
```

### Live Substack API
- About: `https://[handle].substack.com/api/v1/about` — publication info, description, subscriber count
- Posts: `https://[handle].substack.com/api/v1/posts?limit=12` — recent posts
- Both undocumented but stable public endpoints, no auth required
- `fetchSubstackData()` builds a `contextString` for Claude (injected as verified data) and a `posts` array for UI display
- Post URL pattern: `https://[handle].substack.com/p/[slug]`

## API Routes

### POST `/api/research`
- Input: `{ writerName: string, isSelf?: boolean }`
- Output: `WriterProfile` JSON (with `recentPosts` attached if live data fetched)
- Extracts handle from writerName → fetches live Substack data if valid handle found
- Error codes: `INVALID_INPUT` (400), `PARSE_ERROR` (500), `API_ERROR` (500)

### POST `/api/essay-ideas`
- Input: `{ profile: WriterProfile, userNiche?: string, userHandle?: string }`
- Output: `EssayIdeasResult` JSON
- If `userHandle` provided: fetches user's own Substack data as `userData` — separate from inspiration writer's profile
- `effectiveNiche` fallback: `userNiche || profile.writerName`

## Known Limitations / Open Issues

| Issue | Severity | Notes |
|-------|----------|-------|
| No rate limiting | Medium | Any IP can call both APIs unlimited times. Vercel Edge Middleware (IP-based) is the fix for v2 |
| Full profile sent client→server for essay ideas | Low | Client POSTs the full WriterProfile to `/api/essay-ideas`. Fine for now; in v2 store in Vercel KV by session |
| No custom domain | Low | substackr.vercel.app — should be resolved before major public launch |
| Small-writer profile quality | Low | Claude has limited training data on smaller writers. Live fetch helps significantly when handle is provided |
| Anthropic quota shared with OpenClaw | Medium | Monthly spend cap hit once in March 2026. Monitor at console.anthropic.com |
